# Security Policy

## Supported Versions

We actively provide security updates for the following versions of **rails-mcp**:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.** We value your work and want to ensure the security of our users. To report a vulnerability, please use the **Private Vulnerability Reporting** feature:

1.  Navigate to the [Security tab](https://github.com/raja-jamwal/rails-mcp/security) of this repository.
2.  Click on **Vulnerability reporting** under the "Reporting" section.
3.  Click **Report a vulnerability** to submit your findings privately.

We will acknowledge your report within 48 hours and provide a timeline for a fix if the vulnerability is confirmed.

## Our Security Commitment

To ensure the integrity of the project, we implement the following:

* **Immutable Releases:** All releases include SLSA provenance generated via GitHub Actions, allowing you to verify that the code in the release matches the source code in this repository.
* **Dependency Management:** We use Dependabot to monitor for and automatically update vulnerable dependencies.
